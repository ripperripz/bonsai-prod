🚀 DEPLOYMENT STEPS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

STEP 1: LOG IN TO CPANEL
─────────────────────────
1. Go to your hosting provider's cPanel login page
2. Enter your username and password
3. Access the cPanel dashboard

STEP 2: BACKUP EXISTING FILES (IMPORTANT!)
───────────────────────────────────────────
1. In cPanel, click "File Manager"
2. Navigate to public_html/
3. Select all files
4. Click "Compress" → Create ZIP backup
5. Download the backup to your computer

STEP 3: CLEAR PUBLIC_HTML (RECOMMENDED)
────────────────────────────────────────
1. In File Manager, go to public_html/
2. Select all files and folders
3. Click "Delete" (you have a backup!)
4. Confirm deletion

STEP 4: UPLOAD THE ZIP FILE
────────────────────────────
1. In File Manager, ensure you're in public_html/
2. Click "Upload" button (top menu)
3. Click "Select File"
4. Choose: bonsai-cpanel-deployment.zip
5. Wait for upload to complete (may take 5-15 minutes due to size)

STEP 5: EXTRACT THE ZIP FILE
─────────────────────────────
1. After upload completes, close the upload dialog
2. In File Manager, locate bonsai-cpanel-deployment.zip
3. Right-click the ZIP file
4. Select "Extract"
5. Choose "Extract files to: /public_html/"
6. Click "Extract File(s)"
7. Wait for extraction to complete

STEP 6: DELETE THE ZIP FILE
────────────────────────────
1. After extraction, select bonsai-cpanel-deployment.zip
2. Click "Delete"
3. Confirm deletion (saves server space)

STEP 7: VERIFY FILE STRUCTURE
──────────────────────────────
Your public_html/ should now contain:
  ✅ index.html
  ✅ .htaccess (may be hidden - enable "Show Hidden Files")
  ✅ sw.js
  ✅ translations.js
  ✅ assets/ (folder)
  ✅ images/ (folder)

STEP 8: SET FILE PERMISSIONS (IF NEEDED)
─────────────────────────────────────────
1. Select all files in public_html/
2. Click "Permissions" or "Change Permissions"
3. Set:
   - Files: 644 (rw-r--r--)
   - Folders: 755 (rwxr-xr-x)
4. Check "Recurse into subdirectories"
5. Click "Change Permissions"

STEP 9: ENABLE SSL CERTIFICATE
───────────────────────────────
1. In cPanel main page, search for "SSL"
2. Click "SSL/TLS Status"
3. Find your domain
4. Click "Run AutoSSL" or install Let's Encrypt
5. Wait for SSL to activate (may take a few minutes)

STEP 10: TEST YOUR WEBSITE
───────────────────────────
1. Open a new browser tab
2. Visit: https://yourdomain.com
3. Test these pages:
   □ Homepage (English)
   □ https://yourdomain.com/ar (Arabic)
   □ https://yourdomain.com/location
   □ https://yourdomain.com/project
   □ https://yourdomain.com/contact

4. Test functionality:
   □ Language toggle (EN ↔ AR)
   □ Navigation menu
   □ Image loading
   □ Mobile responsiveness
   □ Arabic text on iOS Safari

⚠️  TROUBLESHOOTING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

PROBLEM: "Page not found" when refreshing
SOLUTION: Verify .htaccess file exists in public_html/
          Enable "Show Hidden Files" in File Manager settings

PROBLEM: Images not loading
SOLUTION: Check that images/ folder uploaded correctly
          Verify file permissions are correct (644 for files)

PROBLEM: Website shows old version
SOLUTION: Clear browser cache (Ctrl+Shift+R or Cmd+Shift+R)
          Clear server cache if using caching plugin

PROBLEM: SSL not working
SOLUTION: Wait 5-10 minutes for SSL to propagate
          Check SSL/TLS Status in cPanel
          Contact hosting support if issue persists

PROBLEM: Slow loading
SOLUTION: Verify Gzip/Brotli compression is enabled
          Check .htaccess compression settings
          Test with Google PageSpeed Insights
